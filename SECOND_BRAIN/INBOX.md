# Inbox (Rapid Capture)

Use this during music sessions or coding sessions when you do not want to break flow.

## Capture template

- Date:
- Context: (Sketches / Active / Archive / Settings / Transfer)
- Trigger: what happened
- Friction: what slowed flow
- Idea: smallest possible improvement
- Evidence: (clip, screenshot, file path, repro step)

## Triage rules (daily)

1. If it is immediately buildable: move to `NOW.md`.
2. If it is uncertain and needs discovery: move to `QUESTIONS.md`.
3. If it is a measurable product bet: move to `EXPERIMENTS.md`.
4. If direction is finalized: record in `DECISIONS.md`.
5. If no longer relevant: mark as dropped with one line of reason.

## Intake quality bar

- One friction per line, no essays.
- Include state transition when possible (for example, `Sketches -> Active`).
- Include expected behavior in plain words.
